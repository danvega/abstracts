# SpringOne 2025 Abstracts

A list of sessions I'm submitting to SpringOne 2025

## Spring into Model Context Protocol (MCP)

As Large Language Models transform application development, Spring developers need standardized approaches to connect AI models with external systems. The Model Context Protocol (MCP) provides this standardization for integrating AI capabilities into Spring applications.

We'll demonstrate how to leverage the official Java SDK for MCP (donated by the Spring team) through live coding examples. Starting with basic configuration using Spring's programming model, we'll build custom MCP servers as Spring components and show how to connect Spring applications to various MCP clients.

Attendees will learn what the Model Context Protocol is and how to begin creating their own MCP clients and servers. You'll gain practical insights to immediately improve your Spring-based AI development workflow.

## Beyond REST: Crafting a Modern GraphQL API Live

Skip the slides and dive into code as we build a GraphQL API using Spring Boot. Through live coding, we'll implement schema-first design, batch loading optimization, and query filtering. You'll see how GraphQL solves common REST challenges like over-fetching and endpoint proliferation. We'll explore the latest features from Spring for GraphQL, showcasing recent advancements in the framework. By the end, you'll understand not just GraphQL basics, but production-ready patterns for scalability and performance. Perfect for developers looking to modernize their API architecture with practical, battle-tested approaches.

## Context is King: Building Advanced AI Apps with Spring AI

Large Language Models have transformed application development, but their true potential emerges when properly augmented. This session explores how Spring AI enables developers to overcome LLM limitations through multiple augmentation strategies. You'll discover how to effectively add and manage data in the LLM's context window, build Retrieval Augmented Generation pipelines for grounding responses in your data, integrate tools to extend capabilities beyond text generation, and implement Model Context Protocol for standardized context handling. Through live demonstrations and real-world examples, you'll learn actionable techniques to build more intelligent, reliable AI applications using Spring AI, regardless of your experience level with LLMs.

## Fundamentals of Software Engineering In the age of AI

Agentic coding assistants and editor based AI chat interfaces are altering the development workflow leading some to proclaim the end of software engineering. Is it time to explore other careers? Not so fast, the rumors of our demise are greatly exaggerated! While these tools can boost productivity, to be used effectively, developers still need to master the fundamentals of the software craft. Modern software development demands more than just coding proficiency—it requires navigating an increasingly AI-augmented landscape.

Co-Speaker: Nate Schutta

## The Office Hours Advantage: Building Learning Communities Within Your Org

Discover how to implement the powerful "Office Hours" pattern that Spring Office Hours hosts have refined and seen replicated across organizations. In just 20 minutes, learn our simple, repeatable format for delivering technical updates, sharing news, and creating feedback channels that drive organic knowledge growth. We'll show how this approach not only keeps teams informed but also provides developers with a critical opportunity to hone their presentation skills, one of the most valuable professional abilities they can develop. Leave with a practical blueprint for establishing Office Hours in your own organization, regardless of size or technical focus.

Co-Speaker: DaShaun Carter

## Spring Boot 3.x Rewind: A look back at my favorite features

As Spring Boot 4.0 appears on the horizon, it's the perfect time to celebrate the amazing features that made the 3.x line one to remember. This rapid-fire session highlights the most impactful Spring Boot 3.x innovations that transformed how we build applications. From GraalVM native image support to the migration to Jakarta EE, from improved observability to streamlined developer experience, we'll explore practical implementations of these features with code examples and real-world use cases.

Whether you're a Spring Boot veteran or just getting started, you'll walk away with concrete techniques to immediately leverage these powerful capabilities in your applications. Join me for this fast-paced retrospective as we appreciate the best of what Spring Boot 3.x offered before we embrace the future with 4.0.

## Bootstrap Your Boot: Creating Custom Spring Boot Starters from Scratch

Spring Boot starters simplify application setup by bundling dependencies and auto-configurations. In this session, we'll explore what makes these components so powerful and show you how to create your own custom starters. You'll learn the fundamentals of Spring Boot starter architecture, understand key components like auto-configuration classes and dependency management, and discover best practices for structuring and packaging your starter. We'll move beyond theory with a practical walkthrough of a custom starter I've built and implemented, highlighting real-world applications and common challenges to watch for. Whether you're looking to eliminate boilerplate code, standardize configurations across projects, or simplify complex integrations, you'll leave with the knowledge to build your own starters.

## Grounded AI: Building Spring Applications with Local Open Source Models

While frontier AI models offer impressive capabilities, they come with privacy concerns, data governance challenges, and unpredictable costs. This session demonstrates how to build production-ready Spring AI applications using open source models running locally on your own hardware. You'll learn how to select appropriate models offering multiple capabilities and tool support, implement them in Spring AI applications, and deploy them locally all without sacrificing capability. We'll explore the security, privacy, and cost advantages of this approach while building a working MVP. Leave with practical techniques for leveraging Spring AI with local models that keep your data secure, your costs predictable, and your development velocity high.
